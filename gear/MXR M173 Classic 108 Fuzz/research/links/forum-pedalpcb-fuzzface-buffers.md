REAL-WORLD FUZZ-FACE-AFTER-BUFFER EXPERIENCE (directly relevant: M173 sits behind the Colour Box buffer)
- THE STANDARD SYMPTOM (user BuddytheReow): "A buffer before a FF type circuit will make the FF sound like crap. I've done this by accident and on purpose." -> confirms the thin/wrong-sounding failure mode the M173 risks in this rig.
- COUNTER-EXAMPLE #1 — ACTIVE PICKUPS (user finebyfine): "I play fuzz faces with active pickup guitars all the time and don't mind it at all." Active pickups present a buffered/low-Z source (just like a buffer in front) yet this player is happy. -> EVIDENCE that a buffered source into a FF is a TASTE issue, not an automatic dealbreaker. Reassuring for the M173 behind the VG-800/Colour Box: it can be a valid, usable voice, not "broken."
- COUNTER-EXAMPLE #2 — FF BETWEEN TWO BUFFERS (Marsh schematic): a Fuzz Face sandwiched between buffers "sounds great" per a demo. -> proper buffering on BOTH sides of a FF is a known, good-sounding topology. The M173 effectively does this: buffered Colour Box in front + the M173's own input buffer switch = a "buffered both ways" arrangement that can sound great rather than crap.

RIG TAKEAWAY: The buffer-before-FF rule is real but NOT absolute. Active-pickup players and buffer-sandwiched designs both sound fine. So in Pedalxly the M173 behind the Colour Box (buffer ON) is a legitimate, documented-good configuration — judge it by ear, don't assume it's compromised just because the textbook says "fuzz first."
