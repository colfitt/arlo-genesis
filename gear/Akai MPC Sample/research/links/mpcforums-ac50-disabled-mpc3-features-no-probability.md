AC50-SPECIFIC + AUTHORITATIVE. The clearest community statement of WHY the AC50 lacks features its bigger siblings have. Thread is a feature request for trig **Probability**; the key reply is from MPC-Tutor.

## The headline limitation: no trig Probability on the AC50
OP: "I found one major feature lacking! This would be a Probability feature... this is a basic functionality in all samplers like Elektron Digitakt II, Roland SP-404 MkII, Akai MPC, Novation Circuit Rhythm." (Owner since 2026-05-01.)
- RIG NOTE: the Digitakt 2 and Octatrack in this studio both have conditional/probability trigs; the AC50 does NOT. Don't expect generative trig variation on the MPC — keep that job on the Elektrons. Workaround on the MPC = manual mutes / multiple sequence variations / record live performance variation.

## AUTHORITATIVE answer (MPC-Tutor) — features are deliberately DISABLED, not absent
> "Probability is a feature available on the 'full' standalone MPC sequencer, it's there in the underlying firmware but removed for the MPC Sample, as are many MPC3 features."
> "It's highly unlikely that the MPC Sample is going to get all the missing features found on its bigger siblings, Akai are looking to offer a more simple budget offering, but no doubt if enough people request a feature it's possible they might enable it."
- IMPLICATION: the AC50 runs a cut-down build of the same MPC3-era firmware as the One/Live/X, with many features intentionally stripped — so "the big MPC can do X" does NOT mean the AC50 can. Always verify per-feature; don't assume legacy/full-MPC technique transfers. To lobby for a feature: request via Akai's website.
- Channel to request features: Akai Pro website / support contact form.

## Cross-reference
This is the structural reason behind the other AC50 gaps the community reports: no keygroups, no per-slice editing, no LFOs, chromatic play limited to 16 notes, can't load individual .sqx sequence files, etc. All consistent with "same firmware base, features disabled."
