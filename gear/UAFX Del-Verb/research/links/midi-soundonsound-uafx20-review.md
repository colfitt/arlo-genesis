# Sound On Sound — UAFX 2.0 review (the honest gotchas)

Best source for the PRACTICAL friction of the USB-MIDI workaround.

- **No-preset pedals recall via CC snapshots.** Pedals like Del-Verb "can also now recall saved sounds using MIDI CC snapshots from capable MIDI foot controllers." (No hands-on test of how preset-like it feels — SoS doesn't say it's seamless.)

- **App-page gotcha:** *"the app page defaults to Program Change settings, so you have to select the Control Change page to see the list of available parameters and controller numbers."* For Del-Verb (a no-preset pedal) the CC page is the one that matters — PC is for the preset-saving pedals.

- **USB MIDI has NO THRU.** Quote: *"Unlike traditional DIN-plug based MIDI, USB MIDI has no 'thru' facility."* Therefore: *"you always need a hub to address multiple pedals, since USB MIDI has no thru facility, and a passive hub is recommended here."* (A passive USB hub, not a powered one.)

- **Controllers cited as compatible:** Morningstar MC8 Pro, Disaster Area Designs DMC.micro Gen 4; **CME H2MIDI Pro** as the 5-pin-DIN-to-USB converter.

- **MIDI Clock:** "pedals with tempo-dependent features can now be sync'ed to an external MIDI Clock" — so Del-Verb delay time syncs to clock. (No behavioral test given.)

- **BIGGEST WORKFLOW CAVEAT — can't run the app and MIDI at the same time.** The single USB-C port is shared. To edit voicings in UAFX Control while a MIDI host is plugged in, you must "unplug the pedal from the computer/controller and connect to a phone or tablet instead," then "connect via Bluetooth" for app access. So: tweak voicings over Bluetooth FIRST, then plug into the MIDI host for the gig. You cannot do both live.

TAKEAWAY for the travel/fly role: the MIDI workaround works, but it adds a foot controller + a passive USB hub (for >1 pedal) and a plug-swap dance for app edits. That erodes the "one simple box" appeal — fine in a studio/Apollo setup, heavier for a true carry-on board.
