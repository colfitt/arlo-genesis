A thorough post-1.4 owner review. The most useful real-world gotchas:

## Keybed scanning (power-user fact)
- **10 kHz keybed scanning rate** — fast vs most controllers; the reviewer finds it noticeably more responsive. Relevant for fast synth-bass/arp playing, which is exactly this rig's intended use for the SL.

## GOTCHA — feature buttons open EDIT menus, not the feature
- Pressing **Zones / Sequencer / Scales / Arp** opens that feature's *edit menu*; to actually **enable** the feature you must **hold Shift + press the button**. Unintuitive at first and a recurring source of "why isn't this on?" confusion. (Same complaint as the Elektronauts thread's "I hate holding shift + mode.")

## GOTCHA — Components is browser-bound and browser-picky
- Firmware updates + template editing happen in a **browser window to the Novation site**. The reviewer reports **Firefox does NOT work — use Chrome**. (Practical: keep Chrome around for any Components/firmware session.)

## GOTCHA — Logic control-surface setup is fragile
- Logic integration needs Novation's **control-surface plugin** (the replacement for the old Automap Server). The reviewer had to **trash Logic Pro X's control-surface preferences and reinstall the plugin** before the SL's TFT screens would correctly display Smart Controls data. This is the documented fix path when "Logic parameters don't show on the SL screens." (Novation has an official KB article for exactly this — reset Logic's control-surface prefs.)

## Sequencer visualization limit
- The 16 pads can't show a whole arrangement at once — patterns/steps must be **banked**, even with the rotate options. The 16-step grid is a real visual constraint for longer material.

## Power
- Unlike the MkII, **needs the external PSU** — USB power is insufficient (confirms dossier §8 "no USB power").
