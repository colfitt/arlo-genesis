# SonicFreaks "Microcosm Editor" — Max for Live editor + 31 bundled presets

**THE BIG NEW FACT (corrects on-disk research):** the on-disk dossier + UsageGuide both state the Microcosm
has "NO preset file format and no SysEx dump … presets live on-device only … sharing a preset = publishing the
recipe." That is true for *on-device* presets and for any official Hologram tool, but as of **2024 a third-party
editor exists that DOES store full parameter sets off-device**: the SonicFreaks **Microcosm Editor**, a **Max for
Live device** (Ableton Live 10+ Suite, Max installed). Surfaced via the Gearspace thread post #19 (30 Jul 2024:
"There is a Microcosm Editor available now").

## What it does (verified)
- **"All parameters controllable"** — remotely drives *every* Microcosm parameter over MIDI from inside Ableton.
- **Saves/recalls parameter states as Ableton Live presets** ("click the 'Save' button … give your new preset a
  name"). So a full Microcosm patch (engine + variation + all knob values) can be stored, named, and shared as an
  M4L preset — the practical workaround to the no-SysEx-dump limitation.
- **Looper remote control + Link** — auto-syncs the Microcosm looper start/stop with Ableton's transport.
- **Randomizer** function for generative patch discovery.
- **31 presets included** ("carefully crafted presets as a bonus").

## Honest limits of this source
- **Neither the product page nor the buanzo guide publishes the 31 preset NAMES or their numeric values** — so
  no concrete control-by-control patch can be lifted from here; only the existence + workflow is confirmed.
- It is **Ableton-Suite-only** (needs Max), so it does not help a pure-pedalboard/standalone rig directly — but
  for this rig (records to Apollo, DAW present) it's a legitimate path to back up/recall Microcosm patches.

## Rig relevance
For a rig that prints to a DAW: this is the way to make Microcosm patches *recallable and shareable* despite no
SysEx dump — build the granular bed in the Editor, save as a named M4L preset, and reload exactly. Also exposes
all params for automation (smoother than the coarse hardware encoders, same benefit as EXP/MIDI noted on-disk).
