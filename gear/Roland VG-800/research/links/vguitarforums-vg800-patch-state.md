State of the VGuitarForums VG-800 community re: shared patches. HONEST READ: thin.

KEY THREADS CHECKED:
- "Boss VG-800 Documentation" (topic 38106) — consolidates official manuals (Quick Start,
  Reference, Parameter Guide, MIDI Implementation) + Tone Studio editor manual + Boss
  V-Guitar history (Rod Brakes) + Paul White SoS application guides. Elantric posts the
  key terminology: MEMORY = one patch; LIVESET = an array of memories. Notes Tone Exchange
  downloads aren't auto-saved to the unit — must be exported explicitly. **No actual patch
  files or parameter sets shared in the thread.**
- "Patches" (topic 38204) — turns out to be a GT-1000-Core-to-VG-800 PORTING discussion,
  not patch shares. Will Robinson: "The VG800 has a simpler single signal path for the
  natural pick up" so GT-1000 "kitchen sink" presets do NOT transfer; you rebuild manually
  by matching amp/cab/delay/mod/reverb. **No VG-800 patch files.**
- "FC-300 to control VG-800" (topic 38712) — MIDI control rig discussion, not patches.

UNLIKE older V-Guitars, there is NO dedicated "VG-800 Patch Exchange" sub-board yet. (The
VG-99 and VG-8 each have their own Patch Exchange boards — board 35 and board 122 — full
of user .syx/.tsl files. The VG-800 has only a general-discussion board, board 514, as of
this research.) Community also flags an error in Paul White's article re: alt-tuning on
MIDI output.

TAKEAWAY for the guide: as of mid-2026 the VG-800 user-patch ecosystem is essentially
EMPTY — no mature .tsl liveset library on the forums, no VG-800 Patch Exchange board,
nothing official on Tone Central. The honest answer the dossier predicted (2025 box, no
deep ecosystem yet) is CORRECT. Build from factory presets + Tone Studio, not downloads.

Confidence: HIGH for "no mature shared-patch library yet"; the forum may grow over time.
