# The "set it up ONCE" template — beat the learning curve

Universal community advice for the OT's blank-canvas problem: the machine needs a lot of programming before it's usable, so **build a template Project once, then "load it, go to town, save as new"** every session. Never start from zero.

## What experienced users pre-configure
- **Track 8 = MASTER**, with the master Compressor + a reverb pre-assigned (so you can resample the whole mix and have glue/space ready).
- **A couple of Thru tracks pre-set** for external gear (INAB / INCD already pointed at the right input pairs, DIR = 0) — for this rig, the pedalboard-into-Thru patch lives here permanently.
- **MIDI tracks pre-assigned to the right channels** for your external drum machine / synths (Digitakt, VG-800, etc.) so sequencing other gear is instant.
- **Recording buffers / memory config** set for how you work (dynamic recorders if you do long loops — remember the power-cycle bug after changing mem config).
- **Scene params pre-configured** for the crossfader transition technique (A/B level locks, filter sweeps).
- Sensible **naming + leave many patterns/parts blank** as a playground (and if you share a Set, leave blank canvas for others).

## Organization tips
- Keep a **spreadsheet** of scene + part assignments per template (the OT gives you little on-screen labeling, so an external map keeps you sane).
- Each project can become a different instrument — "standard sampler groovebox," "loop machine," or "mixer/FX processor" — so keep a few purpose-built templates and pick the right one.

## The meta-lesson
The single most repeated piece of curve-taming advice across Elektronauts: **Merlin's Guide first, then build ONE template, then learn ONE concept at a time (Flex → scenes → crossfader → resampling).** Don't try to learn the whole machine at once; it punishes that.
