# Digitakt II — Tips & Tricks (the primary community tips thread)

The single biggest DT2-specific tips thread on Elektronauts. All DT2 unless noted. Distilled high-signal items below (post #/author where captured).

## Top tips by community upvotes

- **Save a trig as a Preset, p-locks and all** (post #9, LyingDalai — most-liked, 62). Hold a trig → press the PRESET/KIT button → "Save new preset." This captures the sound *with all its parameter locks baked in*. Verbatim: "save a trig sound with all its plocks as a new Preset." Huge for the fuzz-wall workflow — lock a crushed/filtered state to a step, then promote it to a reusable preset.

- **Scale-constrained note entry** (post #12, LyingDalai). On TRIG page 1, *push* DATA encoder A (NOTE) and it cycles only through notes of the selected scale (not chromatic). Works while p-locking. There's a setting to make scale-only the default; push the encoder for a chromatic override. Verbatim: "go only through the notes of the selected scale."

- **128-step pattern run at half-resolution / double speed** (post #3, LyingDalai). Treat the 128-step pattern as a 64-step pattern with track Speed x2 → "entering d'n'b territory." Combine: populate with the Euclidean generator, then SHIFT/FUNC + EUC OFF to bake it into normal grid trigs, then drop a GRID machine with random locks (grid 64/32, LEN 4). [DT2-only: 128 steps and the per-track Euclidean generator are DT2 additions.]

- **Fake chords via CTRL-SEL multi-track editing** (post #14, LyingDalai). Use CTRL-SEL to select multiple tracks and edit them simultaneously (same sample, filter, keytrack). Assign different notes per track; engage tracks per trig to build chords across tracks (audio tracks are monophonic, so this is the only way to chord). Lets you e.g. "slide between two notes while the rest stays static."

- **Sample-chain Presets** (post #24, LyingDalai). Assemble 16–32 similar mono samples into a sample chain, save as a Preset for instant recall. "Better [to] get [a] 16 mono sample chain you know well." Pairs with the GRID machine for slicing.

## Other concrete workflow items

- **Page loop / page mute in Grid mode** (post #7, LyingDalai). Hold PAGE → trig keys 1–8 select which page plays; trig keys 9–16 mute pages. "very fast to use" for live arrangement of multi-page patterns.

- **Default kit strategy** (post #24, LyingDalai). Populate sample slots 1–16 with your go-to one-shots so they load by default into every new pattern; save the assembled Kit for reuse across projects.

- **Comb-filter resampling loop** (post #17, Honeysmack). Run loops / single shots through the Comb filter, then resample, then repeat — iterative metallic/resonant degradation. Directly on-aesthetic for turning a banjo pluck into a detuned metallic resonator.

- **De-click envelopes**: to kill clicks on one-shots, set AMP Attack to something slightly above 0 plus a short Decay/Release, and make trig/note lengths the exact step length minus a hair.

- **Manual slicing via Oneshot**: an alternative to GRID — copy a sound to several tracks and nudge the sample START point per track; each becomes its own "slice" with independent pitch/LFO/filter.

- **Extended pitch range** (posts #22–23): tracks transpose ±5 octaves — useful for repitched banjo/baritone one-shots played as melodic instruments.
