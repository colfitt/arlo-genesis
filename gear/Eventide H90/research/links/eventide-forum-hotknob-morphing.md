# HotKnob — morph multiple parameters with one knob/pedal (the analog-feel move)

The HotKnob is the H90's version of the H9's ribbon/expression macro: map a RANGE of multiple parameters to one continuous control, then sweep them together. Unlike a HotSwitch (instant snapshot toggle), the HotKnob is **continuous** — so it's how you do gradual morphs on the H90.

## The three HotKnobs
There are **three HotKnobs: P, A, B.** Each can map multiple parameters across a Preset (or, for P, across the Program).

## Creative morph tricks (from community)
- **Gang them:** map HotKnobs **A and B both to HotKnob P** so one control sweeps parameters in both Presets simultaneously (a true single-knob morph of the whole Program).
- **Expression-pedal the HotKnob:** assign an expression pedal to a HotKnob for foot-controlled morphing.
  - Global (all Programs): **System Settings → Global → Pedal Ctl**, assign the expression pedal to **HotKnob P** (also the trick for using an expression pedal as master Output level).
  - Per-Program: in **Parameters mode, press-and-hold the Perform knob** to assign the HotKnob to an expression pedal for that Program only.

## HotKnob vs HotSwitch (when to use which)
- **HotSwitch** = instant snapshot jump, mutually exclusive, NO gradual change.
- **HotKnob** = continuous sweep of a parameter range; THIS is how you get gradual/morph behavior on the H90 (alongside an expression pedal or a ramped MIDI CC).

## Rig relevance
For slow drone swells, this is the move the gradual-changes thread points to: map a HotKnob to sweep, say, Blackhole Mix + Size together (or a Program-wide P HotKnob ganging both Presets), then drive it with an expression pedal on EXP/CTL — or a ramped CC from a Morningstar/Digitakt. Gives you the timed "bloom into a wall" that a HotSwitch can't do.
