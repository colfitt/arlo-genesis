# Real-world HotSwitch + HotKnob workflow on a small fly-rig H90 (cleaned transcript)

Practical player walkthrough of programming HotSwitches and a HotKnob on a minimal "fly rig" board, using the H90 Control app. The useful, concrete bits:

## The board concept
Small everyday-gig board; H90 + a Lex (Leslie) + a rotating overdrive. The point: the H90 covers most time/space needs so the board stays tiny.

## Navigating: Select vs Perform
- **Select mode**: the encoders scroll presets (e.g. 99, 98, 97...); the green encoder selects. He keeps a "basic preset" at 99 = Tremolo + Reverb for amps that lack onboard trem/verb.
- **Perform mode**: the three footswitches become per-Program controls. First press of Perform = first row (e.g. Reverb on/off, Tremolo on/off); **press Perform a SECOND time = a second row of three more footswitch functions.** So "only three footswitches, but two levels = effectively six functions."

## HotSwitch used as a fixed mix snapshot (his main trick)
- He maps a HotSwitch to the **Reverb Mix** so one footswitch toggles between a "really wet reverb" and a "not-so-wet reverb" — two mix states from one Program.
- Crucially: the HotSwitch he set to change the reverb's wetness **does NOT affect the tremolo at all** — a HotSwitch only moves the parameters you mapped to it.
- Generalizes: "the hot switch can become whatever you want" — a boost (if you have an EQ in a slot), a Leslie slow/fast, more/less delay mix for verse-vs-solo.

## Programming steps he shows (in H90 Control)
1. Build the Program: Preset A = slapback delay (from Preset Library → Favorites/User), Preset B = spring reverb (bypass one to audition).
2. **Control Assignments** tab = the six available footswitches (default active/active/active). Change one to **HotSwitch 1**.
3. The 2nd Perform row can hold e.g. **Tap Tempo** (he notes he could, isn't using it).
4. **Quick Knobs**: he assigns the top three black knobs to live params — e.g. **Wet Mix**, **delay Time**, **Reverb Mix** — for real-time tweaking.
5. Name the Program (e.g. "slap + spring").
6. **HotSwitch 1 → Mix parameter**: press the HotSwitch position, then dial the target value; instant A/B between the two mix states.
7. **HotKnob example**: maps the Preset HotKnob to slapback size/room — sweeps from "almost unhearable little room" up to "big." Press the HotKnob (turns blue), set the max value, save; then save with the knob at the minimum so it starts subtle.

## Player takeaways
- "It takes a while to get used to the platform but it's super powerful."
- Live value: instantly drop a too-much delay/reverb to a subtler mix (verse vs solo) via a HotSwitch, without menu diving.

## Rig relevance
Confirms the practical pattern for the End board: build a Program, map a HotSwitch to a single decisive parameter (reverb/delay Mix, or Blackhole Feedback→Freeze), and put live tweak params on the three Quick Knobs. The "press Perform twice for a second row of three" detail is the cleanest answer to the three-footswitch limit without external switches.
