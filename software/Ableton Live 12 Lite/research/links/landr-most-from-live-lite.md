# Getting the most out of Live Lite — the survival tactics

The best "work within the limits" piece. Four workarounds that directly extend
how far the 8-track ceiling stretches.

## Restating the real limits
- **8 tracks total** (audio+MIDI any combo) vs 16 in Intro.
- **8 mono in / 8 mono out**.
- Fewer stock instruments/effects than Intro — "easily compensated for with the
  countless free plugins on the market."

## Workaround 1 — Racks beat the track limit (the big one)
- "A 16-channel **Drum Rack** only counts as a single track in Lite."
- Same for **Instrument Racks / Audio Effect Racks** — nest many devices and
  parallel chains inside ONE track. The 8-track cap counts top-level tracks, not
  what's inside a rack. This is THE move: build dense multi-layer drones/beats
  inside racks so each only burns one track slot.

## Workaround 2 — Free VST/AU plugins fill the device gaps
- Lite's thin device list (no Operator/Wavetable/Sampler) is offset by free
  3rd-party synths, drum machines, EQs, compressors. Lite **does host VST/AU
  plugins** — so the missing-instruments problem is largely solvable for free.
- For this rig that means the owned AU/VST arsenal (Valhalla, Soundtoys,
  Arturia, Kontakt, etc.) all load in Lite as plugins, sidestepping most stock
  limitations.

## Workaround 3 — Satellite projects + bounce (the track multiplier)
- "Create a project for a drum bus. Layer drum elements with up to eight tracks,
  mix them down, export the result as a stereo drum kit stem." That stem is ONE
  track in the master project.
- Generalizes perfectly to ambient: build a dense 8-track drone "sub-mix" in its
  own set, bounce to a stereo stem, import it as one track in the performance
  set. Repeat → effectively unlimited layers via stem-stacking.

## Workaround 4 — Stock up on samples
- Supplement Lite's library from Freesound / sample marketplaces; load via Simpler
  (which IS in Lite) or Drum Rack.

## Rig takeaway
Racks + plugin hosting + satellite-bounce means the 8-track cap rarely bites for
a hardware-front workflow: the heavy lifting is on hardware, Live just needs a few
clip-launch tracks + rack-dense texture tracks, and final stem-handoff goes to
Logic anyway.
